Implemented MCPV1-3 in this iteration.

### What changed

- `dispatchTool` now returns strict MCP envelopes on both success and failure paths:
  - Success: `{ ok: true, data, meta }`
  - Failure: `{ ok: false, data: null, error, meta }`
  - Unknown tool path is normalized to `tool_not_found` envelope (no throw).
- Threw validation/runtime errors are caught and normalized to envelope `error.code` values (including `invalid_argument`, `internal_error`).
- Token/freshness metadata is moved into `meta`:
  - `toolVersion: "1"`
  - `tokenBudgetUsed`
  - `dataFreshness`
- Added/updated strict output validators for MCP-only tools in dispatch.
- Added focused contract coverage for envelope behavior in interface + engine-contract tests.
